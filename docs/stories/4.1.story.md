# Story 4.1: Command-Line Argument Parsing and Entry Point

**Status:** Done

## Story

**As a** developer,
**I want** a professional CLI interface with standard arguments (--help, --config, --version, --dry-run),
**so that** the system is easy to use, configure, and validate before running in production.

## Acceptance Criteria

1. CLI entry point implemented as `main.py` with main() function
2. Argument parsing using argparse library with all required options
3. Help text includes usage, description, examples, and exit codes
4. Version output format displays correctly
5. Config file validation with appropriate error codes
6. Argument validation with helpful error messages
7. Entry point callable via `python main.py`
8. Shebang support for direct execution
9. Exit code consistency throughout application
10. Unit tests for argument parsing
11. Integration test for argument combinations
12. Manual test verification
